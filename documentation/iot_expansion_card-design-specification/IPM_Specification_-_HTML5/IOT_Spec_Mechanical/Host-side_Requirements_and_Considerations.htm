4.5 Host-side Requirements and Considerations

4.5.1 IoT Connector

IoT Expansion Cards connect to host applications via an IoT Connector (QSFP+) mounted on the host application. This connector style was selected for ease of use, and wide commercial availability from multiple vendors.

The host application can be designed with as many connectors as desired — there is no prescribed limitation on the number of IoT Expansion Cards that a host application may support at one time.

 

Figure 4-7:   IoT Connector (QSFP+ connector)

The QSFP+ connector standard is described in SFF-8436 Specification for QSFP+ 10 Gbs 4X Pluggable Transceiver (Standardized as EIA-964 at Rev 4.8 dated October 31, 2013)

 

Figure 4-8:   Installed IoT Connectors

4.5.2 IoT Expansion Card Mounts

IoT Expansion Cards are mounted on host applications using one of three methods:

•Screwed into spacers

•Screwed into soldered standoffs

•Plugged in via rails

For details, see Mounting Methods.
